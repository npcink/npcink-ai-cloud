Npcink Cloud 隐私政策

本隐私政策说明，当 WordPress 站点管理员通过 Npcink Cloud Addon 插件将站点连接到 Npcink Cloud 时，Npcink Cloud 可能接收哪些信息。

服务域名：https://cloud.npc.ink

我们接收的信息

根据站点配置和功能使用情况，发送到 Npcink Cloud 的请求可能包含：配置的 site identifier、key identifier、request timestamp、nonce、trace identifier、idempotency key、HMAC signature headers、Cloud Base URL target、本地 Npcink 组件提供的 runtime request payload、health check requests、run status 和 run result requests、usage、entitlement 和 observability summary requests、管理员启用后的 metadata-only 插件 monitoring events，以及本地 Npcink 组件启用后的 public content-change hints。

对于由本地主机提供的 media derivative runtime jobs，请求还可能包含短期 source artifact descriptor、derivative parameters，以及本地主机组件提供的 bounded media derivative inputs。

我们不会有意收集的信息

Npcink Cloud Addon 的 metadata-only monitoring 设计为不上传 prompts、generated content、article body content、raw request or response payloads、provider credentials、Cloud API secrets、passwords、cookies、无关系统的 nonces、Authorization headers、database names、table names 或 filesystem paths。

Cloud API Key secret 仅由本地 WordPress 站点用于服务端签名，不会打印到 wp-admin 页面。

可选监控

插件 monitoring 是可选功能，需要管理员明确 opt-in。启用后，monitoring 可能上传 plugin slug、plugin version、event kind、status、timing、error code、route、proposal id、ability id、correlation id、counters 和 latency 等运行元数据。

Monitoring 数据用于提供服务健康、可靠性、质量和诊断摘要。它不是本地 WordPress governance truth。

信息用途

Npcink Cloud 可能使用收到的信息来认证和验证签名请求、提供 hosted runtime 功能、返回 health、entitlement 和 usage summaries、处理 bounded media derivative jobs、提供可选 observability 和 quality summaries、保护服务可靠性、防止滥用并排查服务错误。

数据共享

我们不会出售个人信息。我们可能仅在运营 Npcink Cloud 所需范围内，与基础设施提供商、安全提供商或服务处理商共享有限数据。

数据保留

Operational logs、request metadata、entitlement records 和 monitoring summaries 仅在服务运营、安全、调试、滥用防范、会计或法律要求所需期间保留。

删除请求

站点管理员可以通过 npc@npc.ink 请求删除与账户相关的服务数据。
