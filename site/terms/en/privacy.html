Npcink Cloud Privacy Policy

This Privacy Policy explains what information Npcink Cloud may receive when a WordPress site administrator connects a site to Npcink Cloud through the Npcink Cloud Addon plugin.

Information We Receive

Depending on site configuration and feature use, requests to Npcink Cloud may include configured site identifier, key identifier, request timestamp, nonce, trace identifier, idempotency key, HMAC signature headers, Cloud Base URL target, runtime request payloads supplied by local Npcink components, health check requests, run status and run result requests, usage, entitlement, and observability summary requests, metadata-only plugin monitoring events when enabled by an administrator, and public content-change hints for Site Knowledge synchronization when enabled by local Npcink components.

For host-supplied media derivative runtime jobs, requests may also include a short-lived source artifact descriptor, derivative parameters, and bounded media derivative inputs supplied by the local host component.

Information We Do Not Intentionally Collect

The Npcink Cloud Addon is designed not to upload prompts, generated content, article body content, raw request or response payloads, provider credentials, Cloud API secrets, passwords, cookies, nonces from unrelated systems, Authorization headers, database names, table names, or filesystem paths as part of metadata-only monitoring.

The Cloud API Key secret is used for server-side signing by the local WordPress site and is not printed in wp-admin.

Optional Monitoring

Plugin monitoring is optional and requires explicit administrator opt-in. When enabled, monitoring may upload operational metadata such as plugin slug, plugin version, event kind, status, timing, error code, route, proposal id, ability id, correlation id, counters, and latency.

Monitoring data is used to provide service health, reliability, quality, and diagnostic summaries. It is not local WordPress governance truth.

How We Use Information

Npcink Cloud may use received information to authenticate and verify signed requests, provide hosted runtime features, return health, entitlement, and usage summaries, process bounded media derivative jobs, provide optional observability and quality summaries, protect service reliability, prevent abuse, and troubleshoot service errors.

Data Sharing

We do not sell personal information. We may share limited data with infrastructure providers, security providers, or service processors only as needed to operate Npcink Cloud.

Data Retention

Operational logs, request metadata, entitlement records, and monitoring summaries are retained only as long as needed for service operation, security, debugging, abuse prevention, accounting, or legal requirements.

Deletion Requests

Site administrators may request deletion of account-linked service data by contacting: npc@npc.ink.
