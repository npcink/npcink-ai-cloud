Npcink Cloud Data Retention and Deletion

Npcink Cloud retains service data only for operational, security, debugging, abuse prevention, entitlement, accounting, and legal purposes.

Typical Retention Categories

  • Authentication and request-signing metadata: retained for security and debugging.
  • Runtime request and result metadata: retained for service operation and troubleshooting.
  • Optional monitoring metadata: retained for service health, reliability, quality summaries, and diagnostics.
  • Entitlement and usage summaries: retained for account status, quota, and service access.
  • Media derivative artifacts: retained according to their configured expiration or service policy.
  • Site Knowledge change hints: retained only as needed to deliver public content-change hints to Cloud Site Knowledge systems.

Data Minimization

Npcink Cloud Addon monitoring is metadata-only and is designed not to collect prompts, article body content, generated content, raw payloads, provider credentials, Cloud API secrets, passwords, cookies, Authorization headers, database names, table names, or filesystem paths.

Deletion

Administrators may request deletion of account-linked Npcink Cloud service data by contacting: npc@npc.ink.

Some records may be retained where required for security, abuse prevention, accounting, legal compliance, or backup integrity.

For retention or deletion questions, contact: npc@npc.ink.
