# AI Generated Content Disclosure v1

Status: active.

Reviewed: 2026-06-04.

Purpose: define the minimum disclosure, metadata, logging, and product copy
rules for Magick AI outputs that are generated by, assisted by, or materially
transformed by AI. This is an engineering and product standard, not legal
advice.

## 1. Scope

This standard applies to all Magick AI Cloud surfaces that return text, images,
audio, video, summaries, recommendations, support drafts, analysis, or other
content where an AI model materially contributed to the output.

It applies even when:

- the output is cached
- the output is internal-only
- the output is later edited by a human
- the underlying provider is hidden from the user
- the model is accessed through an OpenAI-compatible adapter

Cloud remains a runtime/service enhancement layer. This standard does not
authorize Cloud to become a WordPress publishing owner, content CMS, prompt
control plane, or article factory.

## 2. Regulatory Baseline

The product posture is "disclose by default." The current baseline is shaped by:

- China `AI-Generated Synthetic Content Labeling Measures`, issued 2025-03-14
  and effective 2025-09-01, requiring service providers to define labeling
  methods/styles in user agreements and guide users on labeling requirements.
- China `Generative AI Services Interim Measures`, effective 2023-08-15,
  requiring certain generated content to be labeled according to deep synthesis
  rules and requiring compliant, lawful, rights-respecting generation.
- EU AI Act Article 50 transparency obligations for generated or manipulated
  content, including user-facing disclosure obligations for certain AI outputs.
- FTC consumer review and testimonial rule, effective 2024-10-21, which treats
  AI-generated fake or false consumer reviews/testimonials as deceptive conduct.

References:

- https://www.cac.gov.cn/2025-03/14/c_1743654684782215.htm
- https://www.cac.gov.cn/2025-03/14/c_1743654685896173.htm
- https://www.gov.cn/zhengce/202311/content_6917778.htm

## 3. Output Classification

Every content-bearing response must classify the output origin:

- `ai_generated`: AI produced the substantive content.
- `ai_assisted`: AI suggested, summarized, transformed, or materially shaped the
  content, but a human may have edited it.
- `rule_generated`: deterministic system text, templates, or metrics with no AI
  model contribution.
- `human_confirmed`: AI output reviewed and approved by a human operator.
- `edited_after_ai`: AI output was changed after generation.

`llm_cached` is still `ai_generated`. Caching changes request cost, not content
origin.

## 4. Required API Metadata

Any API response that returns AI-influenced content must include
`ai_disclosure` next to the content object.

Minimum shape:

```json
{
  "ai_disclosure": {
    "version": "ai-generated-content-disclosure-v1",
    "content_origin": "ai_generated",
    "generated_by_ai": true,
    "ai_assisted": true,
    "visible_label_required": true,
    "visible_label": "AI generated",
    "brand_label": "Magick AI",
    "visible_notice": "Generated by Magick AI. Human review required before use.",
    "review_status": "needs_review",
    "provider_brand_visible": false,
    "machine_readable_required": true,
    "copy_export_notice": "AI generated by Magick AI; human review required.",
    "source_generation_mode": "llm",
    "generated_at": "2026-06-04T00:00:00Z"
  }
}
```

Rules:

- `provider_brand_visible` defaults to `false`; do not expose DeepSeek, OpenAI,
  or other upstream brands to customers unless a separate provider disclosure
  decision approves it.
- `visible_label_required` must be `true` for `ai_generated`, `ai_assisted`,
  `llm`, and `llm_cached`.
- `review_status` must not be `human_confirmed` unless a human confirmation
  event exists.
- `copy_export_notice` must travel with copied, downloaded, emailed, or
  externally surfaced AI content.

## 5. UI Requirements

Visible surfaces must make AI origin obvious at the content block level:

- Show a visible badge such as `AI generated`.
- Show the Magick AI brand marker, not the upstream provider logo.
- Show review state: `Needs human review`, `Edited after AI`, or
  `Human confirmed`.
- For long content blocks, keep an AI label near each independently copyable or
  exportable block.
- Do not hide disclosure only in a tooltip, footer, settings page, or raw log.
- Do not use vague labels such as `Smart`, `Enhanced`, or `Assistant` when the
  content is materially AI-generated.

Recommended default user-facing copy:

- `Generated by Magick AI. Human review required before use.`
- `AI generated - human confirmed.`
- `AI assisted - edited after AI.`

## 6. Copy, Export, And External Delivery

When AI-generated content is copied, exported, emailed, pushed to an external
channel, or prepared for customer-facing delivery:

- preserve `ai_disclosure`
- prepend or append `copy_export_notice` when the destination is plain text
- include machine-readable metadata when the destination supports metadata
- require human review before customer-facing or public use unless the product
  has a separately approved automation policy

For images, audio, and video, future implementations must add both visible and
machine-readable labeling where supported. Text-only outputs currently use
visible label plus API metadata.

## 7. Logging And Audit

Audit logs must retain enough evidence to prove origin and review state without
storing unnecessary prompts or raw model output.

Minimum audit fields:

- `content_origin`
- `generated_by_ai`
- `review_status`
- `generation_mode`
- `provider_id` and `model_id` for internal operator logs only
- `cache_hit`
- `generated_at`
- `confirmed_by` and `confirmed_at` when human confirmation exists

Forbidden audit payloads by default:

- raw prompts
- raw provider payloads
- secrets
- callback bodies
- unnecessary customer content

## 8. Prohibited Patterns

Magick AI must not ship product paths that:

- remove, hide, or falsify required AI labels
- present AI-generated content as human-authored when it is not
- create fake reviews, fake testimonials, fake social proof, fake comments, or
  fake user identities
- claim professional legal, medical, financial, or safety-critical conclusions
  without a separately approved high-stakes compliance review
- auto-publish AI content to WordPress or other public channels from Cloud
- expose upstream provider logos or prices to end customers as if they are the
  purchased product

## 9. Current Implementation

Internal Ops Summarizer responses include `ai_disclosure`:

- deterministic fallback: `content_origin=rule_generated`,
  `visible_label_required=false`
- live LLM output: `content_origin=ai_generated`,
  `visible_label_required=true`, `review_status=needs_review`
- cached LLM output: same AI disclosure as live LLM output, with
  `source_generation_mode=llm_cached`

The admin UI must display Magick AI disclosure near the AI output card and near
independently copyable generated content blocks.

## 10. Release Checklist

Before shipping any new AI content feature:

- classify output origin
- add `ai_disclosure`
- add visible content-level label
- preserve label on copy/export
- define human review state transitions
- add audit fields
- verify provider brand and API price are not customer-visible by default
- verify no prohibited use case is enabled
- update product terms or user-facing instructions when customer-visible AI
  content is introduced
